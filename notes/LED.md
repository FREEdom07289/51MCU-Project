# 51单片机LED控制详解（基于普中A2开发板）

> **核心思想**：不看代码先看图，看懂电流怎么流，代码就是自然推导出来的。

---

## 1. 硬件原理：为什么低电平点亮？

### 1.1 物理连接

在普中A2原理图上，LED模块的连接关系如下：

```
VCC（+5V）
   │
   ├─[R1 限流电阻]─[LED1 正极]─[LED1 负极]─── P2.0
   ├─[R2 限流电阻]─[LED2 正极]─[LED2 负极]─── P2.1
   ├─[R3 限流电阻]─[LED3 正极]─[LED3 负极]─── P2.2
   ...（一直到R8和LED8）...                 ─── P2.7
```

### 1.2 电流方向（水压比喻）

- 电流就像水流，从**高压（VCC=5V）**流向**低压（0V=接地）**
- 单片机引脚 `P2.x` 就像一个"水龙头"：
  - **输出低电平（0V）**：P2.x 相当于接地（0V），电流路径 `5V → 电阻 → LED → P2.x(0V)`，电流流过 → **灯亮** ✅
  - **输出高电平（5V）**：P2.x 也是5V，两端无电压差（5V-5V=0V），电流不流动 → **灯灭** ❌

### 1.3 为什么普中非要这样设计？

51单片机的**"灌电流"能力强**（能吸进去约20mA电流），但**"拉电流"能力弱**（只能吐出来不到1mA）。

LED正常发光需要约3-5mA，所以必须用"低电平点亮"让电流**流进**单片机（灌电流），而不是让单片机流出电流去推LED。

## 2. P2 寄存器：8位大开关盒

`P2.0` 控制 D1，`P2.1` 控制 D2 …… `P2.7` 控制 D8。`P2` 这个寄存器相当于一个8位排插，直接给 `P2` 赋值就能同时控制8个灯。

### 2.1 终极对应表（普中A2专用）

| 二进制（高位→低位） | 对应引脚状态 | 十六进制 | 实际效果（0=亮，1=灭） |
|:---:|:---:|:---:|:---:|
| `1111 1111` | P2.7~P2.0 全为 1 | `0xFF` | 全部熄灭 |
| `0000 0000` | P2.7~P2.0 全为 0 | `0x00` | 全部点亮 |
| `1111 1110` | 仅 P2.0 是 0 | `0xFE` | 只有 D1 亮 |
| `1111 1101` | 仅 P2.1 是 0 | `0xFD` | 只有 D2 亮 |
| `1010 1010` | 偶数位(0,2,4,6)是0 | `0xAA` | D1,D3,D5,D7 亮（奇数列） |
| `0101 0101` | 奇数位(1,3,5,7)是0 | `0x55` | D2,D4,D6,D8 亮（偶数列） |

### 2.2 ⚠️ 新手致命坑

你可能会写 `P2 = 0x0F;` 想让低4位亮（D1-D4）。

算一下：`0x0F = 0000 1111`。位7~4是0（亮），位3~1是1（灭）。
**实际亮的是 D5、D6、D7、D8（高4位）**，跟你以为的完全相反！

> **口诀**：看二进制，哪一位是0，哪一位对应的灯才亮！跟它是在左边还是右边没关系。

## 3. `sbit` 用法：单独控制一个引脚

`sbit` 是 Keil C51 的关键字，用于**位寻址**——单独定义一个引脚变量。

### 3.1 语法格式

```c
sbit 变量名 = 寄存器地址^位号;

// 示例：定义 P2.0 引脚
sbit LED1 = P2^0;   // P2端口的第0位
sbit LED2 = P2^1;   // P2端口的第1位
sbit LED8 = P2^7;   // P2端口的第7位
```

### 3.2 点亮单个LED

```c
#include <reg52.h>

sbit LED1 = P2^0;   // 将P2.0定义为LED1

void main()
{
    LED1 = 0;       // 输出低电平，D1点亮
    // LED1 = 1;    // 输出高电平，D1熄灭
    while(1);       // 让程序停在这里
}
```

> **注意**：`sbit` 只能用于可位寻址的寄存器（如 P0/P1/P2/P3 的各个位），不能随意定义。

## 4. 延时函数：让眼睛"看清"灯的变化

### 4.1 为什么需要延时？

CPU执行一条指令约1微秒。不加延时的话，灯亮灭切换速度太快（百万分之一秒），**人眼视觉暂留**会让你看不到闪烁，只能看到"微微亮"的灯。

### 4.2 延时函数实现

```c
/**
 * @brief 毫秒级延时函数（11.0592MHz晶振，经验值）
 * @param z  延时参数，z=1000 约等于1秒
 */
void delay_ms(unsigned int z)
{
    unsigned int i, j;
    for(i = z; i > 0; i--)       // 外层循环 z 次
        for(j = 110; j > 0; j--); // 内层循环110次约1ms
}
```

> **注意**：这个延时值是经验值，基于11.0592MHz晶振。不同晶振频率需要调整内层循环次数。

## 5. 流水灯实现

### 5.1 方案A：手写移位（不依赖特殊头文件，移植性强）

```c
#include <reg52.h>

void delay_ms(unsigned int z)
{
    unsigned int i, j;
    for(i = z; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main()
{
    unsigned char light = 0xFE;  // 1111 1110 → 只有D1亮（P2.0为0）

    while(1)
    {
        P2 = light;              // 输出当前图案
        delay_ms(100);           // 等100ms

        /*
         * 核心算法：循环左移1位（把0往高位推）
         * 例：1111 1110 → 左移变 1111 1101 (D2亮)
         *
         * light << 1 : 1111 1100  (左移1位，最低位补0)
         * light >> 7 : 0000 0001  (右移7位，把最高位移到最低位)
         * 按位或(|)  : 1111 1101  ✅ 实现循环左移
         *
         * 当 light = 0111 1111 (D8亮) 再左移时：
         * light << 1 = 1111 1110
         * light >> 7 = 0000 0000
         * 按位或得 1111 1110，回到D1亮！实现循环
         */
        light = (light << 1) | (light >> 7);
    }
}
```

### 5.2 方案B：使用 `intrins.h` 库函数（代码更简洁）

```c
#include <reg52.h>
#include <intrins.h>   // 提供 _crol_（循环左移）函数

void delay_ms(unsigned int z)
{
    unsigned int i, j;
    for(i = z; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main()
{
    unsigned char light = 0xFE;

    while(1)
    {
        P2 = light;
        delay_ms(100);
        light = _crol_(light, 1);  // 循环左移1位，等价于手写位运算
    }
}
```

### 5.3 `_crol_` / `_cror_` 说明

| 函数 | 原型 | 作用 | 等价手写 |
|:---:|:---:|:---:|:---:|
| `_crol_` | `unsigned char _crol_(unsigned char c, unsigned char b)` | 将 c 循环左移 b 位 | `(c << b) \| (c >> (8-b))` |
| `_cror_` | `unsigned char _cror_(unsigned char c, unsigned char b)` | 将 c 循环右移 b 位 | `(c >> b) \| (c << (8-b))` |

> **区别**：`_crol_` 直接调用CPU汇编指令 `RL A`，执行速度极快；手写位运算更通用（换STM32也能用）。建议先用方案A理解原理，再偷懒用方案B。

## 6. 交替闪烁（完整示例）

```c
#include <reg52.h>

void delay_ms(unsigned int z)
{
    unsigned int i, j;
    for(i = z; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main()
{
    P2 = 0xFF;  // 初始化：全部熄灭（1111 1111）

    while(1)
    {
        // 第一段：奇数灯亮（D1,D3,D5,D7）
        // 0xAA = 1010 1010，位0/2/4/6是0
        P2 = 0xAA;
        delay_ms(500);

        // 第二段：偶数灯亮（D2,D4,D6,D8）
        // 0x55 = 0101 0101，位1/3/5/7是0
        P2 = 0x55;
        delay_ms(500);
    }
}
```

## 7. 新手常见错误（避坑指南）

| 错误代码/想法 | 硬件的真实反应 | 原因分析 |
|:---|:---|:---|
| `P2 = 0x0F;` 以为D1-D4亮 | 实际亮的是 D5-D8 | `0x0F=0000 1111`，0在左边高位，对应P2.7-P2.4（D5-D8）亮。**看二进制，别只看十六进制尾巴** |
| 直接 `P2 = 0x00;` 然后没延时 | 8个灯看起来微微亮，像没睡醒 | 亮灭切换太快（微秒级），人眼视觉暂留（约50ms），平均亮度降低。必须加延时 |
| 赋值 `P2 = 0xFE;` D1不亮 | 可能P2.0被其他模块占用 | 普中A2有些模块共用引脚（如P2.2-P2.5既接LED又接数码管）。练习时关掉其他模块 |

## 8. 🧪 练手作业

1. **全亮全灭**：写代码让8个灯亮1秒，灭0.5秒，无限循环
2. **呼吸渐变**：让灯从全灭到全亮，再回到全灭。提示：用 `for` 循环把 `0xFF` 逐渐变成 `0x00`
3. **双灯流水**：让两个相邻的灯一起亮，像一对筷子从左扫到右。提示：初始值用 `0xFC (1111 1100)`，循环左移
