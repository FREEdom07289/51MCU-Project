# 51单片机LED控制详解（基于普中A2开发板）

> **核心思想**：不看代码先看图，看懂电流怎么流，代码就是自然推导出来的。

---

## 1. 硬件原理：为什么低电平点亮？

### 1.1 物理连接

普中A2开发板上，8个LED的阳极（正极）通过限流电阻统一接到 VCC（+5V），阴极（负极）分别接到 P2.0 ~ P2.7。

```
VCC（+5V）
   │
   ├─[R1 限流电阻]─[LED1 正极]─[LED1 负极]─── P2.0
   ├─[R2 限流电阻]─[LED2 正极]─[LED2 负极]─── P2.1
   ├─[R3 限流电阻]─[LED3 正极]─[LED3 负极]─── P2.2
   ├─[R4 限流电阻]─[LED4 正极]─[LED4 负极]─── P2.3
   ├─[R5 限流电阻]─[LED5 正极]─[LED5 负极]─── P2.4
   ├─[R6 限流电阻]─[LED6 正极]─[LED6 负极]─── P2.5
   ├─[R7 限流电阻]─[LED7 正极]─[LED7 负极]─── P2.6
   └─[R8 限流电阻]─[LED8 正极]─[LED8 负极]─── P2.7
```

### 1.2 电流方向（水压比喻）

- 电流像水流，从**高压（VCC=5V）**流向**低压（0V=接地）**
- P2.x 引脚就像一个"水龙头"：
  - **输出低电平（0V）**：P2.x 相当于接地，电流路径 `5V → 电阻 → LED → P2.x(0V)`，电流流过 → **灯亮** ✅
  - **输出高电平（5V）**：P2.x 也是5V，两端无电压差（5V-5V=0V），电流不流动 → **灯灭** ❌

### 1.3 为什么必须低电平点亮？

51单片机的**"灌电流"能力强**（能吸进去约20mA），但**"拉电流"能力弱**（只能吐出来不到1mA）。

LED正常发光需要约3-5mA，所以必须用"低电平点亮"让电流**流进**单片机（灌电流）。

## 2. 实验一：点亮一个LED

> 📁 对应代码：`Keil project/01-点亮LED/main.c`

### 2.1 实测代码（逐行注释）

```c
#include <REGX52.H> // 包含 8052 单片机的寄存器和位定义

sbit LED1 = P2^0;   // 定义位变量 LED1 映射到 P2.0

void main()
{
    LED1 = 0;       // 将 P2.0 置 0（低电平），点亮 LED
    while(1)        // 无限循环，保持程序持续运行
    {
        // 空循环：占位，可在此添加延时或外设处理
    }
}
```

### 2.2 关键知识点

| 知识点 | 说明 |
|:---|:---|
| `REGX52.H` | STC89C52 的寄存器头文件，功能等同于 `reg52.h`，但针对 STC 芯片优化 |
| `sbit` | Keil C51 关键字，用于**位寻址**——单独定义一个引脚变量 |
| `LED1 = 0` | 输出低电平→点亮（灌电流）；若写 `LED1 = 1` 则熄灭 |
| `while(1)` | 单片机程序必须死循环，否则会跑飞执行未知代码 |

## 3. 实验二：LED闪烁

> 📁 对应代码：`Keil project/02-LED闪烁/main.c`

### 3.1 实测代码

```c
#include <REGX52.H>

// 延时函数：t=500 约等于500ms（11.0592MHz晶振，经验值）
void Delay(unsigned int t)
{
    unsigned int i, j;
    for(i = t; i > 0; i--)
    {
        for(j = 110; j > 0; j--);
    }
}

void main(void)
{
    while(1)
    {
        P2 = 0x00;   // 全部点亮（0000 0000：8个引脚全为低电平）
        Delay(500);  // 延时500ms — 保持亮的状态
        P2 = 0xFF;   // 全部熄灭（1111 1111：8个引脚全为高电平）
        Delay(500);  // 延时500ms — 保持灭的状态
    }
}
```

### 3.2 为什么需要延时？

CPU执行一条指令约1微秒。不加延时，灯亮灭切换太快（百万分之一秒），**人眼视觉暂留**会让你看不到闪烁，只能看到"微微亮"。

> 💡 代码中保留了早期版本（注释块），用 `sbit LED = P2^0` 控制单个LED闪烁——说明作者经历了从"单灯闪烁"到"整组闪烁"的思考过程。

### 3.3 P2 寄存器：8位大开关盒

直接给 `P2` 赋值就能同时控制8个灯：

| 二进制（高位→低位） | 十六进制 | 实际效果（0=亮，1=灭） |
|:---:|:---:|:---:|
| `1111 1111` | `0xFF` | 全部熄灭 |
| `0000 0000` | `0x00` | 全部点亮 |
| `1111 1110` | `0xFE` | 只有 D1 亮 |
| `1111 1101` | `0xFD` | 只有 D2 亮 |
| `1010 1010` | `0xAA` | D1,D3,D5,D7 亮 |
| `0101 0101` | `0x55` | D2,D4,D6,D8 亮 |

### 3.4 ⚠️ 新手致命坑

你可能会写 `P2 = 0x0F;` 想让低4位亮（D1-D4）。

算一下：`0x0F = 0000 1111`。位7~4是0（亮），位3~0是1（灭）。
**实际亮的是 D5、D6、D7、D8（高4位）**，跟你以为的完全相反！

> **口诀**：看二进制，哪一位是0，哪一位对应的灯才亮！跟它在左边还是右边没关系。

## 4. 实验三：LED流水灯

> 📁 对应代码：`Keil project/03-LED流水灯/`（已重构为3个子目录，各自独立Keil工程）

### 4.1 代码风格进化

在这个实验中，你的代码出现了明显的风格进化：

```c
#include <REGX52.H>
#include <INTRINS.H>

typedef unsigned int u16;    // 类型重定义：缩短类型名
typedef unsigned char u8;

#define LED_PORT P2          // 宏定义端口：方便修改引脚
```

| 技巧 | 作用 | 场景 |
|:---|:---|:---|
| `typedef unsigned int u16` | 把 `unsigned int` 缩写成 `u16` | 代码更简洁，含义更清晰（16位无符号） |
| `typedef unsigned char u8` | 把 `unsigned char` 缩写成 `u8` | 同上（8位无符号） |
| `#define LED_PORT P2` | 给 P2 起别名 | 换引脚时只改一处 |

### 4.2 方案A：手写循环移位（理解底层原理）

> 📁 `03-LED流水灯/手写移位/main-手写移位.c`

```c
#include <reg52.h>

void delay(unsigned int time)
{
    unsigned int i, j;
    for (i = 0; i < time; i++)
        for (j = 0; j < 120; j++);
}

void main(void)
{
    unsigned char x = 0xFE;  // 1111 1110 → 只有D1亮
    unsigned char i;

    while (1)
    {
        // 从左往右扫（D1→D8），循环7次
        for (i = 0; i < 7; i++)
        {
            P2 = x;
            delay(500);
            x = (x << 1) | (x >> 7);  // 循环左移1位
        }
        // 从右往左扫（D8→D1），循环7次
        for (i = 0; i < 7; i++)
        {
            P2 = x;
            delay(500);
            x = (x >> 1) | (x << 7);  // 循环右移1位 ← 新增！
        }
    }
}
```

**`(x << 1) | (x >> 7)` 循环左移解析**：

```
假设 x = 1111 1110 (D1亮)

步骤1: x << 1 = 1111 1100  (左移1位，最低位补0)
步骤2: x >> 7 = 0000 0001  (右移7位，最高位移到最低位)
步骤3: 按位或   = 1111 1101  ✅ (D2亮！)
```

**`(x >> 1) | (x << 7)` 循环右移解析**（你新加的）：

```
假设 x = 0111 1111 (D8亮)

步骤1: x >> 1 = 0011 1111  (右移1位，最高位补0)
步骤2: x << 7 = 1000 0000  (左移7位，最低位移到最高位)
步骤3: 按位或   = 1011 1111  ✅ (D7亮！往回走)
```

> 💡 手写循环右移是你代码中新增的技巧，和左移完全对称——体现了对位运算的深入理解。注释中保留了早期的单向版本，展示了学习过程。

### 4.3 方案B：使用 `_crol_` + `_cror_` 库函数（双向移位）

> 📁 `03-LED流水灯/库函数移位/main-keil头文件.c`

```c
#include <reg52.h>
#include <intrins.h>   // 提供 _crol_ / _cror_ 函数

void delay(unsigned int time)
{
    unsigned int i, j;
    for (i = 0; i < time; i++)
        for (j = 0; j < 120; j++);
}

void main(void)
{
    unsigned char x = 0xFE;
    unsigned char i;

    while (1)
    {
        // 从左往右扫（D1→D8）
        for (i = 0; i < 7; i++)
        {
            P2 = x;
            delay(500);
            x = _crol_(x, 1);  // 循环左移
        }
        // 从右往左扫（D8→D1）
        for (i = 0; i < 7; i++)
        {
            P2 = x;
            delay(500);
            x = _cror_(x, 1);  // 循环右移
        }
    }
}
```

> 注释中保留了早期单向版本 `_crol_`，可以看到你从"单向流水"逐步优化到"双向往复"的完整思考过程。

> **`_crol_` vs `_cror_`**：`_crol_` 循环左移，`_cror_` 循环右移。它们直接调用CPU汇编指令 `RL A` / `RR A`，执行速度极快。

### 4.4 方案C：`typedef` + `#define` 工程风格（最完整）

> 📁 `03-LED流水灯/双向移位/main.c`

```c
#include <REGX52.H>
#include <INTRINS.H>

typedef unsigned int u16;
typedef unsigned char u8;

#define LED_PORT P2

void delay_10us(u16 ten_us)
{
    while(ten_us--);
}

void main()
{
    u8 i = 0;

    LED_PORT = ~0x01;         // ~0x01 = 0xFE，先点亮D1
    delay_10us(50000);        // 初始停顿，方便观察起点

    while(1)
    {
        // 第一段：从左往右扫（D1 → D8）
        for(i = 0; i < 7; i++)
        {
            LED_PORT = _crol_(LED_PORT, 1);
            delay_10us(50000);
        }

        // 第二段：从右往左扫（D8 → D1）
        for(i = 0; i < 7; i++)
        {
            LED_PORT = _cror_(LED_PORT, 1);
            delay_10us(50000);
        }
    }
}
```

**这个版本的精妙之处**：

1. `~0x01` — 按位取反，`0x01 = 0000 0001`，取反得 `1111 1110`，等价于 `0xFE`
2. 两个 `for(i=0; i<7; i++)` — 各移7次（不是8次），因为8次就回到原位了
3. `delay_10us(50000)` — 不同的延时函数风格，用 `while(ten_us--)` 实现，更精简
4. 注释中保留了**方法1（移位法）**，说明你对比过两种实现

## 5. 三种延时函数对比

你的代码中出现了三种延时函数写法：

| 函数 | 文件 | 风格 | 特点 |
|:---|:---|:---|:---|
| `Delay(unsigned int t)` | 1-2LED闪烁 | 双循环，内层 `j=110` | 参数即毫秒数（近似） |
| `delay(unsigned int time)` | 手写移位/Keil头文件 | 双循环，内层 `j=120` | 与上面类似，循环上限不同 |
| `delay_10us(u16 ten_us)` | 流水灯主版 | 单循环 `while(ten_us--)` | 极简，配合 `typedef` 使用 |

> **结论**：没有"唯一正确"的延时函数，只要效果对、你自己能看懂就行。实际项目中常用定时器做精确延时。

## 6. `REGX52.H` vs `reg52.h`

你的前两个实验用 `REGX52.H`，后两个用 `reg52.h`：

| 头文件 | 来源 | 适用芯片 |
|:---|:---|:---|
| `reg52.h` | Keil C51 自带 | 通用8052 |
| `REGX52.H` | STC-ISP 工具提供 | STC89C52 专用，定义更全面（含新增外设寄存器） |

> 对LED实验来说两者无差别。但后续用STC特有功能（如EEPROM、看门狗）时需要用 `REGX52.H`。

## 7. 新手常见错误（避坑指南）

| 错误代码/想法 | 硬件的真实反应 | 原因分析 |
|:---|:---|:---|
| `P2 = 0x0F;` 以为D1-D4亮 | 实际亮的是 D5-D8 | `0x0F=0000 1111`，0在高位，对应P2.7-P2.4亮 |
| 直接 `P2 = 0x00;` 然后没延时 | 灯微微亮，像没睡醒 | 切换太快，人眼视觉暂留（约50ms）导致平均亮度降低 |
| `P2 = 0xFE;` D1不亮 | 可能P2被其他模块占用 | 普中A2的P2.2-P2.5既接LED又接数码管，练习时拔掉无关跳线帽 |

## 9. 练习1：8灯全亮1秒灭0.5秒（修正版）

> 📁 `04-LED练习/练习1/main.c`

### 9.1 修正后的代码

```c
#include <REGX52.H>

void delay(unsigned int t)
{
    unsigned int i, j;
    for (i = t; i > 0; i--)
    {
        for (j = 110; j > 0; j--);
    }
}

void main(void)
{
    while (1)
    {
        P2 = 0x00;      // 8个灯全亮
        delay(1000);    // 延时约1秒

        P2 = 0xFF;      // 8个灯全灭
        delay(500);     // 延时约0.5秒
    }
}
```

### 9.2 常见错误回顾

你之前的错误版本（[按键/main.c](../Keil%20project/按键/main.c) 中仍在）：
```c
unsigned char x = P2;  // ❌ x是RAM普通变量，不是P2寄存器！
x = 0x00;              // ❌ 修改x无法改变硬件引脚电平
```

| 错误 | 原因 | 修正 |
|:---|:---|:---|
| `unsigned char x = P2;` | x只是拷贝了P2上电瞬间的值，存在RAM里 | 直接操作 `P2 = 0x00;` |
| 延时 `int i,j;` | 延时全为正数，有符号int浪费 | 用 `unsigned int i,j;` |

## 10. 练习2：双灯流水灯

> 📁 `04-LED练习/练习2/main.c`

### 10.1 完整代码

```c
#include <reg52.h>
#include <intrins.h>   // _crol_ 循环左移

void delay_ms(unsigned int t)
{
    unsigned int i, j;
    for(i = t; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main()
{
    unsigned char i;

    while(1)
    {
        unsigned char x = 0xFC;  // 1111 1100 → D1、D2亮

        // 循环7次（不是8次！第8次会变成单灯）
        for(i = 0; i < 7; i++)
        {
            P2 = x;
            delay_ms(200);
            x = _crol_(x, 1);    // 两个0同时左移
        }
    }
}
```

### 10.2 执行逐帧动画

| 步数 | pattern 二进制 | 十六进制 | 亮灯 |
|:---:|:---|:---:|:---|
| 初始 | `1111 1100` | `0xFC` | D1、D2 |
| 1 | `1111 1001` | `0xF9` | D2、D3 |
| 2 | `1111 0011` | `0xF3` | D3、D4 |
| 3 | `1110 0111` | `0xE7` | D4、D5 |
| 4 | `1100 1111` | `0xCF` | D5、D6 |
| 5 | `1001 1111` | `0x9F` | D6、D7 |
| 6 | `0011 1111` | `0x3F` | D7、D8 |
| → 跳出循环 | 重置为 `0xFC` | `0xFC` | 回到 D1、D2 |

### 10.3 为什么循环7次不是8次？

第8次左移：`0011 1111` → `0111 1110`，只剩一个0 → **筷子断了**！所以只跑7步，然后靠 `while(1)` 外层重置 `x = 0xFC` 回到起点。

### 10.4 速度调整实验

```c
delay_ms(200);  // 匀速
delay_ms(50);   // 快速 → 像震动模式
delay_ms(500);  // 慢速 → 像老爷散步
```

修改 `i < 7` 为 `i < 6` 试试——筷子走到中间就折返了（留给按键交互练习）。
