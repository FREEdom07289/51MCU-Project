# 51单片机工具库与头文件

> 整理 Keil C51 开发中常用的头文件、类型定义和编程模式。

---

## 1. 头文件对比：`REGX52.H` vs `reg52.h`

你的项目同时使用了两者：

| 头文件 | 来源 | 适用芯片 | 你的使用位置 |
|:---|:---|:---|:---|
| `reg52.h` | Keil C51 自带 | 通用8052/8051 | 1-3流水灯（手写移位、Keil头文件） |
| `REGX52.H` | STC-ISP 工具提供 | STC89C52专用 | 01-点亮LED、02-LED闪烁、03-流水灯主版 |

### 对LED实验的影响

两者在 LED 实验层面**完全等价**，都定义了 `P2`、`P2^0` 等寄存器。区别在于 `REGX52.H` 额外定义了 STC 特有外设寄存器（EEPROM、看门狗、PCA等），后续高级实验时会用到。

### 头文件的核心作用

把单片机内部寄存器的**物理地址**映射为C语言变量名：

```c
// reg52.h / REGX52.H 内部大致做了这些事：
sfr P0 = 0x80;   // P0端口 → 地址0x80
sfr P1 = 0x90;   // P1端口 → 地址0x90
sfr P2 = 0xA0;   // P2端口 → 地址0xA0（LED控制用这个！）
sfr P3 = 0xB0;   // P3端口 → 地址0xB0

// 以及可位寻址的位：
sbit P2_0 = P2^0;
sbit P2_1 = P2^1;
// ... 以此类推
```

> 没有这些头文件，你的代码里就不能写 `P2 = 0xFE;`，而是要写 `*(unsigned char *)0xA0 = 0xFE;`。

## 2. `intrins.h` — 内联汇编函数库

### 2.1 常用函数

| 函数 | 对应汇编 | 作用 | 示例 |
|:---|:---:|:---|:---|
| `_crol_(c, b)` | `RL A` | 将 c **循环左移** b 位 | `_crol_(0xFE, 1)` → `0xFD` |
| `_cror_(c, b)` | `RR A` | 将 c **循环右移** b 位 | `_cror_(0x7F, 1)` → `0xBF` |
| `_nop_()` | `NOP` | 空操作，延时1个机器周期 | 精确时序控制 |
| `_irol_(i, b)` | — | 16位循环左移 | — |
| `_iror_(i, b)` | — | 16位循环右移 | — |

### 2.2 你的项目中的实际使用

```c
// 在 1-3LED流水灯/main.c 中：
#include <INTRINS.H>

// 从左往右扫：每次循环左移1位
LED_PORT = _crol_(LED_PORT, 1);   // D1→D2→...→D8

// 从右往左扫：每次循环右移1位
LED_PORT = _cror_(LED_PORT, 1);   // D8→D7→...→D1
```

### 2.3 `_crol_` 等价手写

```c
// 这两行完全等价：
x = _crol_(x, 1);
x = (x << 1) | (x >> 7);    // 你的 手写移位.c 用的就是这个
```

> **建议**：先手写理解原理，再用 `_crol_` 简化代码。你的项目里两个版本都保留了，这是很好的学习习惯。

## 3. `typedef` — 类型重定义（你的编码习惯）

你的流水灯主版代码中使用了类型重定义：

```c
typedef unsigned int u16;    // 16位无符号整数
typedef unsigned char u8;    // 8位无符号整数
```

### 为什么这样写？

| 好处 | 说明 |
|:---|:---|
| **简洁** | `u8` 比 `unsigned char` 少打很多字 |
| **语义清晰** | `u16` 一看就知道是16位，"u"=unsigned |
| **便于移植** | 换个平台只需改 typedef，不用全文替换 |

### 使用示例

```c
void delay_10us(u16 ten_us)   // 参数类型一目了然：16位无符号
{
    while(ten_us--);
}

void main()
{
    u8 i = 0;                 // 循环变量：8位够用（0~255）
    // ...
}
```

## 4. `#define` — 宏定义端口（你的编码习惯）

```c
#define LED_PORT P2   // 给 P2 起个别名
```

### 好处

换开发板时只需改一处：

```c
// 普中A2：LED接P2
#define LED_PORT P2

// 如果换成另一个板子，LED接P1：
#define LED_PORT P1    // ← 只改这一行，下面所有代码自动生效
```

## 5. 延时函数合集（来自你的项目）

你的5个源文件中出现了三种延时函数：

### 5.1 标准双循环延时（毫秒级）

```c
// 来源：02-LED闪烁/main.c
void Delay(unsigned int t)
{
    unsigned int i, j;
    for(i = t; i > 0; i--)
        for(j = 110; j > 0; j--);  // 内层110 ≈ 1ms（11.0592MHz经验值）
}
// 用法：Delay(500); → 约500ms
```

### 5.2 双循环变体

```c
// 来源：main-手写移位.c / main-keil头文件.c
void delay(unsigned int time)
{
    unsigned int i, j;
    for (i = 0; i < time; i++)
        for (j = 0; j < 120; j--);  // 内层120，稍微不同的经验值
}
// 用法：delay(500);
```

### 5.3 精简单循环延时

```c
// 来源：1-3LED流水灯/main.c
void delay_10us(u16 ten_us)
{
    while(ten_us--);   // 极简写法，适合粗略延时
}
// 用法：delay_10us(50000); → 约500ms
```

> 💡 以上都是**软件延时**（让CPU空转），实际时间取决于晶振频率。精确计时需要用定时器（后续课程会学）。
