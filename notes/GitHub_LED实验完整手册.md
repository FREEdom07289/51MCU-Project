## 一、字节、位换算基础

### 1. 最小单位 bit（位）

1 bit = 1个二进制0/1，单片机引脚单独控制1bit（如 `P2^0`）。

### 2. 基础换算公式

- 1 Byte（字节）= 8 bit（位）
- 拓展存储单位：1KB = 1024Byte、1MB = 1024KB

### 3. 51常用数据类型（字节/取值范围）

| 类型 | 占用字节 | bit数 | 取值范围 | 适用场景 |
|:---|:---|:---|:---|:---|
| `unsigned char` | 1Byte | 8bit | 0~255 | IO端口、段码、短延时、循环计数 |
| `unsigned int` | 2Byte | 16bit | 0~65535 | 长延时、超大数值计数 |
| `int`（有符号） | 2Byte | 16bit | -32768~32767 | 单片机极少使用 |

### 4. 进制权重（8位char）

8位bit权重：128 64 32 16 8 4 2 1

示例：`00000011` = 2+1=3；`11111111` = 255

### 5. 溢出规则

- `unsigned char a = 255; a = a + 1;` → a = 0
- `unsigned int b = 65535; b = b + 1;` → b = 0

### 6. 代码实例区分

```c
// char仅最大255，延时不足
unsigned char i, j;

// int最大65535，适合长延时
unsigned int i, j;
```

### 7. IO端口字节概念

P0/P1/P2/P3每组端口 = 1字节（8bit），对应8个引脚；`sbit` 单独操作单bit。

## 二、LED硬件底层基础知识

### 1. LED基础物理特性

发光二极管（LED），单向导电；长脚=阳极，短脚=阴极；阳极电压＞阴极电压才发光。

### 2. 限流电阻硬性要求

LED必须串联220Ω/470Ω电阻，否则电流过大烧毁LED与单片机IO口。

### 3. 两种硬件驱动接法（普中51使用共阳极）

**① 共阳极（普中开发板默认）**

LED阳极统一接5V，阴极经电阻接单片机IO；
电平逻辑：IO输出0（低电平）→灯亮；IO输出1（高电平）→灯灭。

**② 共阴极（极少使用）**

LED阴极接GND，阳极接IO；IO输出1亮，输出0灭。

### 4. 51单片机IO驱动特性

吸入电流（灌电流）能力强，输出电流（拉电流）弱，因此开发板统一采用共阳极电路。

### 5. IO位定义与端口操作

- `sbit led = P2^0;`：单独控制P2第0引脚LED
- `P2 = 0x00;`：P2端口8个LED全亮；`P2 = 0xFF;` 全灭

### 6. 单LED标准闪烁代码

> 【可直接编译下载】

```c
#include <reg52.h>
sbit led = P2^0;

void Delay(unsigned int t)
{
    unsigned int i, j;
    for(i = t; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main(void)
{
    while(1)
    {
        led = 0;       // 低电平点亮
        Delay(500);    // 延时约500ms
        led = 1;       // 高电平熄灭
        Delay(500);
    }
}
```

### 7. LED常见故障排查

1. LED不亮：电平写反、拨码开关未开启、hex未下载、硬件断路
2. LED微弱发光：IO浮空，硬件电路异常
3. LED烧毁：缺失限流电阻

## 五、P2端口作为特殊功能寄存器

### 1. P2寄存器核心定义

P2是地址 `0xA0` 的8位特殊功能寄存器（SFR），头文件内部定义 `sfr P2 = 0xA0;`，占用1字节8bit；

寄存器bit位与引脚一一对应：D7~D0 ↔ P2.7~P2.0；

寄存器bit=0 → 引脚低电平；bit=1 → 引脚高电平。

### 2. 两种寄存器操作方式

**① 单bit操作 `sbit`**

仅修改寄存器其中1位，其余bit数值不变：

```c
sbit led = P2^0;
led = 0;  // 仅改变P2.0引脚电平
```

**② 整体端口赋值**

一次性重写8位全部bit，控制全部8路LED：

```c
P2 = 0x00;  // 全亮
P2 = 0xFF;  // 全灭
```

### 3. 单LED常亮完整拆解代码

> 【可直接编译下载】

```c
#include <reg52.h>
sbit led = P2^0;

void main(void)
{
    led = 0;     // 修改P2寄存器bit0=0，引脚低电平，LED点亮
    while(1);    // 死循环锁住寄存器数值，防止程序跑飞篡改电平
}
```

逐行解析：

1. `#include <reg52.h>`：引入寄存器定义，识别P2硬件寄存器
2. `sbit led = P2^0;`：给P2最低bit0单独命名，精准单引脚控制
3. `led = 0;`：硬件电流通路：5V → LED阳极 → 阴极 → 排阻 → P2.0低电平，形成回路点亮
4. `while(1);`：无死循环程序会顺序跑飞，P2寄存器数值被改写，灯光异常

### 4. 8灯同步闪烁代码

> 【可直接编译下载】

```c
#include <reg52.h>

void Delay(unsigned int t)
{
    unsigned int i, j;
    for(i = t; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main(void)
{
    while(1)
    {
        P2 = 0x00;   // 8bit全0，全部LED点亮
        Delay(500);
        P2 = 0xFF;   // 8bit全1，全部LED熄灭
        Delay(500);
    }
}
```

## 六、`void main(void)` 括号内void说明

### 1. Keil 51单片机环境

`void main()` 与 `void main(void)` 编译运行效果完全一致，均可正常生成hex文件。

### 2. 标准C语言语法区别

1. `void main(void)`：标准严谨写法，明确声明函数无任何输入参数，禁止传参
2. `void main()`：空括号在标准C中代表参数未知，语法不规范，仅编译器兼容简写

### 3. 其他函数区分

1. 无参延时标准写法：`void Delay(void)`，调用只能写 `Delay();`
2. 带参函数不可写void：`void Delay(unsigned int t)`

### 4. 代码书写建议

作业、规范代码使用 `void main(void)`；日常简写 `void main()` 不影响运行；禁止省略返回值写 `main()`，会触发编译器警告。

## 七、LED单灯闪烁完整详解

### 1. 闪烁核心原理

循环交替修改P2寄存器对应bit的0/1电平，搭配软件延时锁住电平时长，人眼识别亮灭交替。

### 2. 完整闪烁代码

> 【可直接编译下载】

```c
#include <reg52.h>
sbit led = P2^0;

void Delay(unsigned int t)
{
    unsigned int i, j;
    for(i = t; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main(void)
{
    while(1)
    {
        led = 0;       // 低电平点亮
        Delay(500);    // 保持亮约500ms
        led = 1;       // 高电平熄灭
        Delay(500);    // 保持灭约500ms
    }
}
```

### 3. 延时函数作用

单片机运算速度极快，无延时切换电平仅微秒级，肉眼无法区分亮灭；双层for空循环消耗CPU时间，固定电平持续时长；t数值越大，延时越长，闪烁速度越慢。

### 4. 关键语法区分

- `while(1);` 带分号：空循环，内部代码仅执行一次，LED常亮无闪烁
- `while(1){代码}`：大括号包裹逻辑，无限循环实现闪烁

### 5. 寄存器底层变化流程

1. `led = 0` → P2寄存器bit0=0，引脚低电平，灯亮；延时期间寄存器数值固定
2. `led = 1` → P2寄存器bit0=1，引脚高电平，灯灭；延时期间寄存器数值固定
3. 循环往复，bit0持续切换0/1，产生闪烁视觉效果

## 八、8灯同步闪烁

1. `P2 = 0x00` 二进制 `00000000`：P2寄存器全部8bit置0，所有引脚输出低电平，8个LED同步点亮
2. `P2 = 0xFF` 二进制 `11111111`：P2寄存器全部8bit置1，所有引脚输出高电平，8个LED同步熄灭
3. 和单灯闪烁核心区别：
   - `sbit`：仅修改单个bit，控制单灯
   - `P2 = 十六进制数`：一次性重写完整8bit，同步控制全部8路LED
4. 补充：十六进制前缀 `0x` 不可省略；`0X00` 与 `0x00` 大小写编译器无区分

## 九、LED流水灯全套代码+原理

### 前置规则

P2寄存器bit=0灯亮；`1 << num` 左移生成单bit高电平；`~` 按位取反适配共阳极硬件。

### 1. 单向基础流水灯（P2.0 → P2.7）

> 【可直接编译下载】

```c
#include <reg52.h>

void Delay(unsigned int t)
{
    unsigned int i, j;
    for(i = t; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main(void)
{
    unsigned char num;
    while(1)
    {
        for(num = 0; num < 8; num++)
        {
            P2 = ~(1 << num);   // 移位+取反，逐个点亮
            Delay(400);
        }
    }
}
```

#### 移位运算解析

`1 = 00000001`，`1 << num` 左移后仅第num位为1；`~` 取反后仅第num位为0，对应单灯点亮。

- num=0 → `~00000001 = 11111110`（P2.0亮）
- num=7 → `~10000000 = 01111111`（P2.7亮）

### 2. 双向往复流水灯

> 【可直接编译下载】

```c
#include <reg52.h>

void Delay(unsigned int t)
{
    unsigned int i, j;
    for(i = t; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main(void)
{
    unsigned char num;
    while(1)
    {
        // 左到右
        for(num = 0; num < 8; num++)
        {
            P2 = ~(1 << num);
            Delay(400);
        }
        // 右到左（跳过两端避免重复）
        for(num = 6; num > 0; num--)
        {
            P2 = ~(1 << num);
            Delay(400);
        }
    }
}
```

### 3. 奇偶间隔流水灯

> 【可直接编译下载】

```c
#include <reg52.h>

void Delay(unsigned int t)
{
    unsigned int i, j;
    for(i = t; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main(void)
{
    while(1)
    {
        P2 = 0xAA;   // 10101010 偶数引脚亮
        Delay(400);
        P2 = 0x55;   // 01010101 奇数引脚亮
        Delay(400);
    }
}
```

### 4. 流水灯常见故障

1. 多灯同时亮：遗漏 `~` 按位取反，电平逻辑相反
2. 流动速度过快：增大延时函数参数
3. 仅运行单次停止：流水代码未放入 `while(1){}` 无限循环

## 十、单片机6大基础位运算知识点

适用对象：P0/P1/P2/P3 8位寄存器（bit7~bit0对应引脚7~0），普中硬件bit=0灯亮。

### 1. 按位左移 `<<`（流水灯核心）

格式：数值 `<<` n；所有二进制位左移n位，右侧补0，高位溢出丢弃。

示例：`1 << 0 = 00000001`；`1 << 3 = 00001000`；配合 `~` 实现单灯点亮。

### 2. 按位取反 `~`

所有bit翻转，1变0、0变1；适配共阳极电路，将单bit高电平转为低电平点灯。

示例：`~00000001 = 11111110`。

### 3. 按位右移 `>>`

数值 `>>` n；所有位右移n位，左侧补0；用于反向流水、字节拆分。

### 4. 按位与 `&`（清零指定bit）

规则：同bit全1才为1，否则0；单独清零某一位，不改变其余电平。

示例：`P2 = P2 & 0xFE;` 强制最低bit0置0，P2.0灯亮。

### 5. 按位或 `|`（置1指定bit）

规则：同bit有1则1；单独将某一位置1，关闭对应LED。

示例：`P2 = P2 | 0x01;` 最低bit0置1，P2.0灯熄灭。

### 6. 按位异或 `^`（翻转bit电平）

规则：同bit数值相同=0，不同=1；一键切换灯亮灭，极简闪烁。

示例：`P2 ^= 0x01;` 每次执行翻转P2.0电平。

### 复合位运算符（简写）

`<<=`、`>>=`、`&=`、`|=`、`^=`；`P2 ^= 0x01` 等价 `P2 = P2 ^ 0x01`。

### 易错点

1. 移位/取反仅运算不改变原值，必须赋值给寄存器才生效
2. `unsigned char` 仅8位，`1 << 8` 溢出直接等于0
3. `sbit` 属于语法，不属于位运算

## 十一、`typedef`/`#define`/`_crol_` 循环移位流水代码详解

### 完整代码

> 【可直接编译下载】

```c
#include <REGX52.H>
#include <INTRINS.H>   // 提供 _crol_ / _cror_ 循环移位函数

// typedef：数据类型重定义，简化书写
typedef unsigned int u16;
typedef unsigned char u8;

// #define 宏定义：端口文本替换
#define LED_PORT P2

// 简易10微秒延时
void delay_10us(u16 ten_us)
{
    while(ten_us--);
}

void main(void)
{
    u8 i = 0;
    LED_PORT = ~0x01;        // 初始化仅P2.0点亮
    delay_10us(50000);       // 停留约500ms

    while(1)
    {
        // 循环左移：从P2.0流到P2.7
        for(i = 0; i < 7; i++)
        {
            LED_PORT = _crol_(LED_PORT, 1);
            delay_10us(50000);
        }
        // 循环右移：从P2.7流回P2.0
        for(i = 0; i < 7; i++)
        {
            LED_PORT = _cror_(LED_PORT, 1);
            delay_10us(50000);
        }
    }
}
```

### 模块拆分讲解

**1. 头文件**

- `<REGX52.H>`：寄存器端口定义
- `<INTRINS.H>`：内置循环移位库函数；`_crol_` 循环左移、`_cror_` 循环右移，移出位自动补到另一端

**2. `typedef` 类型重定义**

- `u16 = unsigned int`：16位无符号整型，延时计数
- `u8 = unsigned char`：8位无符号字符，端口、循环计数
- 优势：缩短代码长度，统一数据类型书写

**3. `#define LED_PORT P2` 宏定义**

编译前全文本替换，所有 `LED_PORT` 等价 `P2`；修改硬件端口仅需改一行，便于项目维护。

**4. 延时函数 `delay_10us(u16 ten_us)`**

单层空循环软件延时，`ten_us = 50000` ≈ 500ms。

**5. `_crol_` / `_cror_` 循环移位逻辑**

- `_crol_(变量, 1)`：整体8bit左移1位，最高位移到最低位
- `_cror_(变量, 1)`：整体8bit右移1位，最低位移到最高位
- 基于当前P2寄存器数值移位，实现来回双向流水；普通 `<<` 移位每次重新生成单灯电平，无法回流。

### 注释屏蔽的基础移位流水写法

```c
// 被注释掉的单向移位写法（供对比参考）
// for(i = 0; i < 8; i++)
// {
//     LED_PORT = ~(0x01 << i);   // 每次从零生成单路亮灯
//     delay_10us(50000);
// }
```

每次从零生成单路亮灯，仅单向流动，无法直接回流。

## 十二、宏定义 `#define` 完整讲解

### 1. 基础概念

`#define` 是预处理指令，编译程序前执行纯文本替换，不属于运行代码；格式：`#define 宏名 替换内容`。

### 2. 代码示例：端口宏

```c
#define LED_PORT P2
LED_PORT = ~0x01;
// 预处理替换后等价：P2 = ~0x01;
```

优势：硬件端口修改仅需改动宏定义一行，无需全局替换代码，可读性更强。

### 3. 宏定义三大分类

**① 常量/端口无参宏（单片机最常用）**

```c
#define LED_PORT P2
#define DELAY_TIME 50000
```

**② 带参数宏（简易类函数）**

```c
#define SQUARE(x) (x)*(x)
// 调用 SQUARE(3) → 替换为 (3)*(3)
```

**③ 条件开关宏（项目调试）**

```c
// #define TEST
#ifdef TEST
    // 测试代码
#endif
```

### 4. 宏、typedef、变量核心区别

| 特性 | `#define` | `typedef` | 变量 `u8 i;` |
|:---|:---|:---|:---|
| 时机 | 编译前文本替换 | 编译阶段类型别名 | 运行阶段分配内存 |
| 内存 | 无分配 | 无分配 | 分配内存存储数值 |
| 类型检测 | 无 | 编译器识别 | 严格类型 |

### 5. 高频易错点

1. 宏定义末尾**禁止加分号**
   - 错误：`#define LED_PORT P2;` → 替换后生成语法错误 `P2; = ~0x01;`
2. 带参宏参数必须包裹括号，避免运算优先级错误
3. 宏全局生效，无局部作用域
4. `#define` 仅文本替换，无类型校验；`typedef` 严格区分数据类型

## 十三、练习1：8灯亮1秒灭5秒 代码纠错与修正

### 原错误代码问题

```c
// 错误代码：操作普通变量而非硬件寄存器
void main(void)
{
    unsigned char x = P2;
    while(1)
    {
        x = 0x00;        // x是RAM变量，不是P2寄存器！
        delay(1000);
        x = 0xFF;
        delay(500);
    }
}
```

#### 三大核心错误

1. `unsigned char x = P2;`：x是RAM普通变量，仅拷贝上电瞬间P2数值，修改x无法改变硬件P2寄存器，LED无任何变化；必须直接操作 `P2`
2. 延时时长颠倒：需求亮1s、灭5s，原代码亮1000、灭500时长相反
3. 延时函数 `int i, j;` 规范问题：延时计数全为正数，推荐使用 `unsigned int` 避免有符号溢出bug

### 修正后完整可运行代码

> 【可直接编译下载】

```c
#include <REGX52.H>

void delay(unsigned int t)
{
    unsigned int i, j;
    for(i = t; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main(void)
{
    while(1)
    {
        P2 = 0x00;     // 8灯全亮
        delay(1000);   // 持续约1秒
        P2 = 0xFF;     // 8灯全灭
        delay(5000);   // 持续约5秒
    }
}
```

### 逻辑说明

1. `P2 = 0x00`：寄存器8bit全0，全部LED点亮，延时1000≈1秒
2. `P2 = 0xFF`：寄存器8bit全1，全部LED熄灭，延时5000≈5秒
3. `while(1)`：无限循环，持续「亮1s → 灭5s → 亮1s → …」往复

### 延时适配说明

基于11.0592MHz晶振，内层 `j = 110` 为经验参数；时长偏差可微调延时函数传入数值。

## 十四、练习2：呼吸渐变灯（全灭→全亮→全灭）代码纠错与标准实现

### 需求提示

使用for循环，公式 `P2 = 0xFF - i`；普中共阳极：数值越大灯越暗，数值越小灯越亮。

### 原错误代码问题

1. 第二段渐变循环逻辑错误：`for(i = 0; i < 0xFF; i++){ P2 = i; }`，重新从0递增，灯光渐变出现断层，无法从全亮平滑回到全灭
2. 旧代码 `P2 = 0x00 + x` 逻辑反向：x增大P2数值增大，灯光变暗，无法实现渐亮效果

### 标准正确呼吸渐变代码

> 【可直接编译下载】

```c
#include <REGX52.H>

void delay(unsigned int t)
{
    unsigned int i, j;
    for(i = t; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main(void)
{
    unsigned char i;
    while(1)
    {
        // 阶段1：全灭 缓慢渐变到全亮（i从0→255）
        for(i = 0; i < 0xFF; i++)
        {
            P2 = 0xFF - i;   // 数值递减 = 越来越亮
            delay(10);       // 延时控制渐变速度
        }
        // 阶段2：全亮 缓慢渐变到全灭（i从255→0）
        for(i = 0xFF; i > 0; i--)
        {
            P2 = 0xFF - i;   // 数值递增 = 越来越暗
            delay(10);
        }
    }
}
```

### 渐变逻辑拆解

1. **渐亮段** i=0→255：`P2 = 0xFF - i`，P2从0xFF（全灭）逐步降低至0x00（全亮）
2. **渐暗段** i=255→0：`P2 = 0xFF - i`，P2从0x00（全亮）逐步升高至0xFF（全灭）
3. **速度调节**：`delay(10)` 数值越小渐变越快，数值越大灯光变化越柔和

### 硬件补充说明

普通51 IO无硬件PWM，该代码为视觉延时渐变，非硬件高精度PWM呼吸灯；两层for循环放入 `while(1)` 实现无限往复呼吸效果。

> ⚠️ 关于PWM呼吸灯（单个LED亮度渐变）：需要用定时器中断或微秒级延时实现占空比调节。由于51单片机无DAC，纯软件PWM会阻塞CPU，实际项目中不推荐。此内容留到定时器章节后用硬件PWM实现。

## 附录：故障排查汇总

### Keil 编译类

| 现象 | 原因 | 解决方法 |
|:---|:---|:---|
| 无法生成hex文件 | 未勾选 Create HEX File | 魔法棒 → Output → 勾选 Create HEX File |
| 编译报 Error | 语法错误（缺少分号、括号不匹配等） | 双击错误信息跳转到对应行修正 |
| 编译报 Warning | 非致命警告（如未使用变量） | 不影响hex生成，可忽略或修正 |
| 路径含中文报错 | Keil不支持中文路径 | 将工程移到纯英文路径下 |
| `undefined identifier P2` | 未包含头文件 | 添加 `#include <reg52.h>` 或 `<REGX52.H>` |

### STC-ISP 烧录类

| 现象 | 原因 | 解决方法 |
|:---|:---|:---|
| 下载失败 / 一直等待 | 未冷启动 | 先点"下载/编程"，再给开发板上电 |
| 烧录后现象不对 | 烧了旧的hex文件 | 每次修改代码后，在STC-ISP中重新打开最新hex |
| 串口无法识别 | 驱动未安装或USB线问题 | 安装CH340驱动，换一根数据线（非充电线） |
| 烧录成功但LED不亮 | 拨码开关未打开 | 检查LED模块旁的拨码开关是否拨到ON |

### 代码逻辑类

| 现象 | 原因 | 解决方法 |
|:---|:---|:---|
| LED全亮而非单灯亮 | 遗漏 `~` 按位取反 | `P2 = (1 << num)` 应改为 `P2 = ~(1 << num)` |
| 灯只亮一次不循环 | 代码未放入 `while(1){}` | 将流水灯/闪烁逻辑放入死循环大括号内 |
| 灯常亮不闪烁 | 写了 `while(1);` 带分号 | 分号导致空循环，应使用 `while(1){ }` 大括号 |
| 修改x变量灯不变 | 操作了普通变量而非P2寄存器 | 直接写 `P2 = 0x00;`，不要写 `x = 0x00;` |
| `P2 = 0x0F;` 亮的灯不对 | 0=亮1=灭，`0x0F=00001111` | 高4位是0所以亮D5-D8，要亮D1-D4请用 `0xF0` |
| 流水灯速度太快/太慢 | 延时参数不合适 | 调大/调小 `Delay()` 的传入参数 |
| `1 << 8` 不亮 | unsigned char只有8位，移位8次溢出为0 | 限制移位次数 0~7 |

### 硬件类

| 现象 | 原因 | 解决方法 |
|:---|:---|:---|
| 全部LED不亮 | 拨码开关未开启 / 排阻松动 | 检查LED模块拨码开关、排阻RP9/RP10 |
| 单个LED不亮 | 该LED损坏或焊接问题 | 换一个P2引脚测试，确认是LED还是IO口问题 |
| LED微亮 | IO口浮空（未初始化电平） | 在while(1)之前初始化 `P2 = 0xFF;` |
| LED烧毁 | 缺少限流电阻 | 检查排阻RP9/RP10是否正常（各470Ω） |
