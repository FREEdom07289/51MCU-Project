# 51单片机按键输入

> 学完LED（输出），进入按键（输入）——从"让单片机说话"到"让单片机听你说话"的关键一步。

---

## 1. 按键基础原理

### 1.1 按键是什么？

- 按键就是一个**机械开关**
- 按下时电路导通 → 单片机引脚读到低电平（0）
- 松开时电路断开 → 单片机引脚读到高电平（1）

### 1.2 为什么需要消抖？

- 按键内部是**金属弹簧片**，按下瞬间在接触和断开间弹跳几次
- 弹跳持续时间：**5-20ms**
- 比喻 🏓：乒乓球落地弹几下才停——单片机听到了每次弹跳声，以为你按了好多次

---

## 2. 普中A2按键硬件

| 按键 | 引脚 | 按下电平 |
|:---|:---|:---|
| K1 | P3.1 | 低电平（0） |
| K2 | P3.0 | 低电平（0） |

> IO口有内部上拉电阻，默认状态为高电平（1），所以初始状态 `LED1 = 1` 熄灭。

## 3. 实战一：按键控制LED流水灯（方向+启停）

> 📁 `Keil project/按键/05-按键/main.c` | 日期：2026-07-08

### 3.1 功能说明

| 按键 | 功能 |
|:---|:---|
| **K1**（P3.1） | 切换流水方向（D1→D8 ⇄ D8→D1） |
| **K2**（P3.0） | 启停流水灯（运行 ⇄ 暂停） |

### 3.2 完整代码

```c
/*
 * 05-按键控制LED流水灯 - 普中A2开发板
 * 功能：K1切换LED流水方向 / K2启停流水灯
 * 硬件连接：
 *   LED（8个） → P2.0 ~ P2.7（低电平点亮）
 *   独立按键K1 → P3.1（按下为低电平）
 *   独立按键K2 → P3.0（按下为低电平）
 */

#include <REGX52.H>
#include <INTRINS.H>

// ===== 类型重定义 =====
typedef unsigned int u16;
typedef unsigned char u8;

// ===== 引脚定义 =====
#define LED_PORT P2
sbit KEY1 = P3^0;   // K2：切换流水方向
sbit KEY2 = P3^1;   // K1：启停流水灯

// ===== 全局变量 =====
u8 direction = 0;   // 0=左移(D1→D8)，1=右移(D8→D1)
u8 running   = 1;   // 1=流水运行中，0=暂停

// ===== 函数声明 =====
void delay_ms(u16 ms);
u8   key_scan(void);

// ===== 主函数 =====
void main()
{
    u8 led = 0xFE;              // 初始：1111 1110 → 只有D1亮

    while(1)
    {
        u8 key = key_scan();    // 扫描按键

        if(key == 1)            // K2 → 切换方向
            direction = !direction;

        if(key == 2)            // K1 → 启停切换
            running = !running;

        if(running)             // 运行状态才执行流水
        {
            LED_PORT = led;
            delay_ms(200);

            if(direction == 0)
                led = _crol_(led, 1);  // 左移
            else
                led = _cror_(led, 1);  // 右移
        }
    }
}
```

### 3.3 消抖函数 `key_scan()`

```c
u8 key_scan(void)
{
    if(KEY1 == 0)           // ① 检测按下
    {
        delay_ms(20);       // ② 等20ms消抖
        if(KEY1 == 0)       // ③ 再次确认
        {
            while(KEY1 == 0); // ④ 等松手（防连击）
            return 1;         // ⑤ 返回键值
        }
    }

    if(KEY2 == 0)           // K2同理
    {
        delay_ms(20);
        if(KEY2 == 0)
        {
            while(KEY2 == 0);
            return 2;
        }
    }

    return 0;               // 无按键
}
```

### 3.4 架构亮点

| 设计 | 说明 |
|:---|:---|
| `direction` 全局变量 | 0/1 控制左移/右移，`!direction` 一键翻转 |
| `running` 全局变量 | 1/0 控制运行/暂停，不改变led当前值 |
| `key_scan()` 独立函数 | 封装消抖逻辑，主循环只关心键值 |
| `while(KEY1 == 0);` | 等待松手，防止按一次触发几十次 |

## 4. 实战二：按键控制LED翻转（最简消抖）

> 📁 `Keil project/按键/06-独立按键控制LED亮灭/main.c` | 日期：2026-07-15

### 4.1 功能说明

K1 每按一次，D1 状态翻转（亮→灭→亮→灭…）。

### 4.2 完整代码

```c
/*
 * 06-独立按键控制LED亮灭 - 普中A2开发板
 * 功能：K1按键每按一次，LED状态翻转
 * 硬件连接：
 *   LED（D1） → P2.0（低电平点亮）
 *   独立按键K1 → P3.1（按下为低电平）
 */

#include <REGX52.H>
#include <intrins.h>

sbit KEY1 = P3^1;
sbit LED1 = P2^0;

// 按键消抖延迟（@11.0592MHz）
void Delay20ms(void)
{
    unsigned char i, j;
    for (i = 0; i < 20; i++)
        for (j = 0; j < 200; j--);
}

void main(void)
{
    LED1 = 1;               // 初始状态：熄灭（高电平）

    while (1)
    {
        if (KEY1 == 0)      // ① 检测按下
        {
            Delay20ms();    // ② 消抖
            if (KEY1 == 0)  // ③ 再次确认
            {
                LED1 = !LED1;       // ④ 翻转LED状态
                while (KEY1 == 0);  // ⑤ 等待松手
                Delay20ms();        // ⑥ 松手后再消抖
            }
        }
    }
}
```

### 4.3 错误总结（代码注释原文）

你的代码末尾记录了实践中踩过的3个坑：

| 错误 | 原因 | 后果 |
|:---|:---|:---|
| 消抖延时不够 | 抖动5-20ms，延时太短滤不掉 | 按键误触发 |
| 没写松手等待 | `while(KEY1 == 0);` 缺失 | 按一次触发多次 |
| LED初始状态错误 | IO有上拉电阻，默认高电平 | 第一次按下效果反了 |

> 💡 这段注释是你从"犯错"到"理解"的成长记录，比正确代码更有价值。

## 6. 常见错误回顾：`x = P2` 陷阱

你的按键初版代码（现已废弃）曾犯过这个错误：

```c
unsigned char x = P2;  // ❌ x是普通RAM变量，拷贝了P2的值
x = 0x00;              // ❌ 修改x不影响硬件引脚
```

| 错误 | 正确 |
|:---|:---|
| `x = 0x00;` × | `P2 = 0x00;` ✅ |
| 操作的是RAM变量 | 操作的是地址0xA0的硬件寄存器 |

> 💡 **核心认知**：`P2` 不是普通变量——它是硬件寄存器，地址 `0xA0`。`unsigned char x = P2;` 只是把上电瞬间的值拷贝到RAM，后续修改 `x` 和硬件完全无关。这是嵌入式编程最基础也最重要的概念。
