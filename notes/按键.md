## 1. 已知要点（来自DeepSeek对话摘要）

### 1.1 按键是什么？

- 按键就是一个**机械开关**
- 按下时电路导通 → 单片机引脚读到低电平（0）
- 松开时电路断开 → 单片机引脚读到高电平（1）

### 1.2 为什么需要消抖？

- 按键内部是**金属弹簧片**，按下瞬间不是稳稳闭合，而是在接触和断开间弹跳几次
- 弹跳持续时间：**5-20ms**
- 单片机很灵敏，会把每一次弹跳都当作一次"按下"
- 比喻 🏓：乒乓球落地弹几下才停——单片机听到了每次弹跳声，以为你按了好多次

### 1.3 消抖方案

- **软件消抖**：检测到按下后，延时20ms再检测一次，确认状态稳定后再执行动作
- **硬件消抖**：使用电容滤波或RS触发器（在普中A2上不常用）

## 2. 软件消抖模板代码

```c
#include <reg52.h>

sbit KEY1 = P3^0;   // 假设按键接在P3.0 [待核实普中A2实际引脚]
sbit LED1 = P2^0;

void delay_ms(unsigned int z)
{
    unsigned int i, j;
    for(i = z; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main()
{
    while(1)
    {
        if(KEY1 == 0)           // ① 检测到按键按下（低电平）
        {
            delay_ms(20);       // ② 延时20ms，等抖动过去
            if(KEY1 == 0)       // ③ 再次确认按键是否仍然按下
            {
                LED1 = ~LED1;   // ④ LED状态翻转
                while(KEY1 == 0); // ⑤ 等待按键松开（防止按一次触发多次）
            }
        }
    }
}
```

### 2.1 消抖流程说明

```
按键按下 → 等待20ms消抖 → 再次确认 → 执行动作 → 等待松开 → 回到检测
   │                         │
   └─ 抖动期（忽略）         └─ 稳定期（确认有效）
```
