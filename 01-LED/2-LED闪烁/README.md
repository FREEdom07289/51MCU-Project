# LED闪烁实验

> 普中A2开发板 · STC89C52RC · 11.0592MHz晶振

---

## 一、功能说明

控制开发板上的 LED0 以固定频率**交替亮灭**，实现闪烁效果。本实验在"点亮一个LED"的基础上引入**软件延时**概念，是学习单片机时序控制的入门实验。

---

## 二、硬件连接

| 元件 | 单片机引脚 | 说明 |
|------|-----------|------|
| LED0（板载） | P2.0 | 普中A2开发板板载LED，共阳极接法 |

### 共阳极 LED 驱动原理

```
VCC（+5V）
  │
  ├──▶ 限流电阻 ──▶ LED（正极）
  │                  │
  │               P2.0（负极/控制端）
  │
  └─ 当 P2.0 = 0（低电平）→ 电流流通 → LED 点亮
     当 P2.0 = 1（高电平）→ 无电流   → LED 熄灭
```

- **低电平点亮**：P2.0 输出 0 时，LED 两端形成电压差，电流从 VCC 经 LED 流向引脚，LED 发光
- **高电平熄灭**：P2.0 输出 1 时，LED 两端电压相等，无电流通过，LED 熄灭

> ⚠️ 不同开发板的 LED 接法可能不同！如果发现亮灭逻辑相反，将代码中 `LED = 0` 和 `LED = 1` 互换即可。

上电后 LED0 以约 **1Hz** 的频率闪烁（亮约 500ms → 灭约 500ms → 重复）。

## 四、完整代码

```c
#include <reg52.h>

sbit LED = P2^0;    // 定义LED引脚为P2.0

void Delay(void)     // 软件延时函数（约500ms）
{
    unsigned int i, j;
    for(i = 0; i < 200; i++)
        for(j = 0; j < 200; j++);
}

void main(void)
{
    LED = 1;         // 初始化为熄灭状态

    while(1)         // 主循环
    {
        LED = 0;     // 点亮LED
        Delay();     // 延时保持
        LED = 1;     // 熄灭LED
        Delay();     // 延时保持
    }
}
```

## 五、逐行精讲

### 5.1 头文件 `#include <reg52.h>`

```c
#include <reg52.h>
```

`reg52.h` 是 Keil C51 编译器提供的**标准头文件**，定义了 51 单片机的所有特殊功能寄存器（SFR）。

打开这个头文件，你会在 Keil 安装目录下看到类似这样的定义：

```c
// reg52.h 内部片段（示意）
sfr P0 = 0x80;   // P0口寄存器，地址0x80
sfr P1 = 0x90;   // P1口寄存器，地址0x90
sfr P2 = 0xA0;   // P2口寄存器，地址0xA0
sfr P3 = 0xB0;   // P3口寄存器，地址0xB0
```

**sfr（Special Function Register）** 关键字将寄存器名称映射到具体的硬件地址。有了这个头文件，我们才能用 `P2` 这个名字来操作 P2 端口。

> 📌 有些例程使用 `#include <REGX52.H>`，两者功能相同，前者是旧版命名，后者对应 STC89C52。本项目统一使用 `<reg52.h>`。

---

### 5.2 位定义 `sbit LED = P2^0;`

```c
sbit LED = P2^0;
```

`sbit` 是 Keil C51 的**扩展关键字**，用于定义**可位寻址**的特殊功能寄存器中的某一位。

| 关键字 | 作用 | 示例 |
|--------|------|------|
| `sfr` | 定义一个8位特殊功能寄存器 | `sfr P2 = 0xA0;` |
| `sbit` | 定义 SFR 中的某一位 | `sbit LED = P2^0;` |

`P2^0` 中的 `^` **不是异或运算符**，而是 Keil C51 编译器规定的位地址表示法，表示 P2 端口的第 0 位（最低位）。

P2 端口的 8 个位完整定义：

```c
sbit P2_0 = P2^0;   // 第0位（最低位）
sbit P2_1 = P2^1;   // 第1位
sbit P2_2 = P2^2;   // 第2位
sbit P2_3 = P2^3;   // 第3位
sbit P2_4 = P2^4;   // 第4位
sbit P2_5 = P2^5;   // 第5位
sbit P2_6 = P2^6;   // 第6位
sbit P2_7 = P2^7;   // 第7位（最高位）
```

定义 `sbit LED = P2^0;` 之后，代码中就可以直接用 `LED` 这个名字来控制 P2.0 引脚，而不需要每次都写 `P2^0`。

> 💡 **为什么要用 sbit？** 因为直接操作整个 P2 端口（如 `P2 = 0xFE;`）会影响 P2 口的全部 8 个引脚。使用 `sbit` 可以**精确控制某一个引脚**，不影响其他引脚的状态。

### 5.3 延时函数 `Delay()`

```c
void Delay(void)
{
    unsigned int i, j;
    for(i = 0; i < 200; i++)
        for(j = 0; j < 200; j++);
}
```

#### 工作原理

这是单片机编程中最简单的延时方法——**软件延时**（Software Delay），也称为**忙等待**（Busy Wait）。

**核心思路**：让 CPU 执行大量"无意义"的循环指令，消耗 CPU 时间，从而达到延时的效果。

#### 延时时间估算

在 11.0592MHz 晶振下，51 单片机的机器周期为：

```
机器周期 = 12 / 晶振频率 = 12 / 11.0592MHz ≈ 1.085μs
```

嵌套循环的执行过程：
- **外层循环**：200 次
- **内层循环**：200 次
- **总循环次数**：200 × 200 = **40,000 次**

每次内层循环包含几条汇编指令（比较、自增、跳转），大约需要几个机器周期。粗略估算：

```
总延时 ≈ 40,000 × 几个机器周期 × 1.085μs ≈ 200~500ms
```

> ⚠️ **重要**：这个延时值**不精确**，会受以下因素影响：
> 1. **编译优化等级**：Keil 的优化选项会改变循环的汇编实现
> 2. **晶振频率**：不同晶振下的机器周期不同
> 3. **数据类型**：`unsigned int` vs `unsigned char` 的循环开销不同
>
> 精确延时应该使用**定时器/计数器**，这将在后续 `04-定时器` 模块中学习。

#### 软件延时 vs 硬件延时

| 对比维度 | 软件延时（Delay循环） | 硬件定时器延时 |
|----------|----------------------|---------------|
| 实现方式 | CPU空循环 | 片内定时器硬件计数 |
| CPU占用 | 100%（无法做其他事） | 低（可同时处理其他任务） |
| 精确度 | 低，受编译和晶振影响 | 高，精确到微秒 |
| 代码复杂度 | 简单 | 需要配置寄存器 |
| 适用场景 | 简单演示、非关键时序 | 精确时序、多任务系统 |

### 5.4 主函数 `main()`

```c
void main(void)
{
    LED = 1;         // ① 初始化：熄灭LED

    while(1)         // ② 无限循环
    {
        LED = 0;     // ③ 点亮
        Delay();     // ④ 保持点亮状态
        LED = 1;     // ⑤ 熄灭
        Delay();     // ⑥ 保持熄灭状态
    }
}
```

#### ① 初始化状态

```c
LED = 1;  // 熄灭LED
```

上电后，51 单片机的 I/O 口默认输出高电平。`LED = 1` 显式设置 P2.0 为高电平，确保 LED 初始为熄灭状态。

> 💡 即使不写这一行，LED 默认也是熄灭的。但**显式初始化是良好习惯**——代码意图清晰，不受默认状态变化的影响。

#### ② `while(1)` 无限循环

```c
while(1)
{
    // 循环体永远执行
}
```

单片机的程序**一般不退出**。与 PC 程序不同，单片机没有操作系统可以"返回"，main 函数退出后程序会跑飞或复位。

`while(1)` 是嵌入式系统的**标准写法**，等价于：
```c
for(;;) { }   // 另一种写法，效果完全相同
```

#### ③~⑥ 亮灭循环

```c
LED = 0;     // P2.0输出低电平 → LED点亮
Delay();     // CPU空转约500ms，LED保持点亮
LED = 1;     // P2.0输出高电平 → LED熄灭
Delay();     // CPU空转约500ms，LED保持熄灭
// 回到③，周而复始
```

这四个步骤构成一个完整的**闪烁周期**：

```
亮（LED=0）→ 延时 → 灭（LED=1）→ 延时 → 亮（LED=0）→ ...
```

## 七、修改闪烁频率

通过修改 `Delay()` 函数的循环次数可以改变闪烁频率：

```c
// 快速闪烁（约5Hz）—— 减少循环次数
void Delay(void)
{
    unsigned int i, j;
    for(i = 0; i < 50; i++)      // 原来是200
        for(j = 0; j < 50; j++); // 原来是200
}

// 慢速闪烁（约0.5Hz）—— 增加循环次数
void Delay(void)
{
    unsigned int i, j;
    for(i = 0; i < 500; i++)     // 原来是200
        for(j = 0; j < 500; j++);// 原来是200
}
```

> ⚠️ 注意：`unsigned int` 的范围是 0~65535，循环次数不要超过这个范围。如需更长延时，可以嵌套更多层循环。

## 十、常见问题

<details>
<summary><b>Q1: LED不闪烁，一直亮或一直灭？</b></summary>

**原因排查**：
1. 延时太短，肉眼无法察觉闪烁 → 增大 Delay 循环次数
2. 开发板 LED 是共阴极接法 → 将代码中 0 和 1 互换
3. 程序未正确烧录 → 重新用 STC-ISP 下载
4. 看门狗复位（本程序未涉及）
</details>

<details>
<summary><b>Q2: 延时时间不准确？</b></summary>

软件延时本身就是不精确的，详见 5.3 节。如需精确定时，使用定时器（后续课程）。
</details>

<details>
<summary><b>Q3: 什么时候用 while(1)，什么时候用 for(;;)？</b></summary>

两者完全等价，编译后的汇编代码也相同。`while(1)` 更直观，使用更广泛。选择符合团队习惯的写法即可。
</details>
